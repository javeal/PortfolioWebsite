Facial Recognition Website Login

This project is intended to be an interface for a facial recognition project I worked on in college. This was my first ever machine learning project! This was also my final project in my digital signal processing class described in the coursework section. It is also mentioned in the past projects section. My goal is to clean up the code I wrote 2 years ago and improve the speed. The facial recognition algorithm makes use of PCA to reduce the size of the data and remove noise. The object then follows the PCA demensionality reduction with a tweaked version of k-nearest nieghbors (KNN). In this version the reference distance is the average face and the number of nearest neighbors is 1. I know much more about PCA and KNN now which should allow me to improve the project significantly! I would like to build a web based interface using the Django framework that will allow users to interact with the algorithm. I want the users to be able to save data to the webserver and login to view their data using their face. This is an ambitious project that will require frontend work that could be challenging for me! Some deliverables for this project are given below

    Requirements:
  • The website will support uploading of images to train the algorithm based on new users
  • The webiste will have a backup login system in case facial recognition fails
  • The website will have all data collection set to optional
  • The website will login users in a timely manner
  • The website will access the webcam to allow user to take pictures and upload them
  • The website must have a threshold of error that minimizes incorrect classifications
  • The website will have a way to report incorrect classifications

Relevant Coursework

Machine Learning EE425

This class was taken in the spring of 2019, and taught by Dr. Namrata Vaswani. The course was an introduction to machine learning that was primarily focused on implimentation of algorithms. The class consisted of a lecture that covered the theory behind the algorithms while the homework was to impliment the algorithms in python. We usually used the MNIST dataset for our assignments. Some of the topics we covered in this class include, logistic Regression, gradient decent, naive bayes, maximum likelyhood estimation, support vector machines, principle component analysis, and clustering. The course included a few lectures on the basics of neural networks and the kalman filter, however no assignments we given on these topics.

Digital Signal Processing

This class was taken in the spring of 2018 and taught by Dr. Alexsandar Dodganzich. This was an introductory course to digital signal Processing that covered basic signal fundamentals for the first half and DSP techniques for the second half. In the first half of the course we focues on the Fourier Series, DTFT, Z transform, and the DFT. In the second half of the course we covered things like windowing, the FFT algorithm, filter design, sampling and reconstruction and image processing. This course included a lab that was done in MATLAB. Lectures were theoretical with problem sets given as assignments.

Communication Systems

This class was taken in the fall of 2018 and taught by Dr. Sang Kim. This course covered the theory of communication systems. This class did not have a lab but included MATLAB portions of the homework. This class covered topics such as FM, AM, and other modulation schemes. Some other topics were line coding, signal properties, signal transmissions, sampling and reconstruction.

Responsive Vehicle

Description:

The objective was to navigate through an obstacle course, without being able to look at the obstacle course. The project was done using a Rumba Vaccum fitted with a Texas Instruments Micro-Controller and a variety of sensors. This project was coded in C and made use of GUI to navagate the robot. Commands were sent to the robot through the connection once data had been recieved by the GUI. Some of the obstacles included cliffs, tall poles, short poles and an out of bounds line.

DTMF Encoder/Decoder

Description:

DTMF is the logic of encoding frequencies for dial up calls. The various dial tones are a combination of of two frequencies, and those frequencies depend on the location of the number on the dial pad. This project required the design of very narrow band pass filter that could decode the frequencies the system recieved. The signals were incoded by the generation of sinusoids based on the number entered. The system produced the tones and delivered the decoded results. This was done using MATLAB.

Skills:

Filter Design and MATLAB

DTMF encoder/decoder Report

Random Number Generator

Description:

The Random Number Generator is a project developed using Altera products in the area of digital circuit design. The numbers were generated using a formula that produced a output that varied depending on the input. The design process included creating state machines, truth tables, and circuit diagrams in altera. For convience, some of the components were coded using Verilog. Some components inside the Generator were Register Files, Registers and a parallel shift Register. The input was controlled with switches on an Altera board, and the output was shown on 7 segment displays.

Skills:

Digital Circuit Design, Boolean Logic and Verilog

Sound System Receiver

Description:

This project can be consisted of the design of 3 analog filters. There was a second order active low pass filter, a second order active high pass filter, and a first order active band pass filter. The idea of the filters was to divide parts of the signal to send to different types of speakers. For example, low frequencies would be directed to a subwoofer. Some of the components used to create this system were op amps, capacitors, inductors and resistors.

Skills:

Analog Circuit Design, Circuit Analysis, Filter Design and Testing
